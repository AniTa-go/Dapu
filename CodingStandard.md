# 常用的8条代码规范

```

1.代码要严格的缩进对齐，函数之间适当的用空行隔开。

2. 名字规范: 名字要易懂，单词之间用大写字母来连接(不能用下划线链接)，不能有任何的缩写，不要担心太长。
- 好的：SetBackgroundColor， 不好的SetBkgdColor(缩写)，不好的set_background_color(下划线)
- 绝对不允许，i,j,k之类的没有意思的名字出现

3. 名字规范2
- 文件名以大写字母开头
- 类名以大写字母开头
- 变量名以小写字母开头
- 函数名以大写字母开头

4. 每个代码文件必须有统一的版权申明
// Copyright 2011 Kiwi Island Studio. All rights reserved.

5.代码必须正式严谨，不能提交随意注释掉的临时代码。

6. 一个函数，一个类只干一件事情
- 函数不能太长，一般不超过100行。长函数应该拆分.
- 一行不能太宽，宽度一般不超过100

7. 不能有神奇的数字，数字应该赋给一个有意思的常量名。
8. 整个工程编译不能有警告（warning）

其他具体参考Google Java Style Guide

```
